# Where Does the Model Come From? (Data vs Existing Model)

## Which model file does the app use? (when there are many files in `models/`)

The app **only** loads from **two fixed filenames**, in this order:

1. **`final_forthesis_bidirectionallstm_and_encoders_exercise_classifier_model.keras`** – tried first (Keras 3 format).
2. **`final_forthesis_bidirectionallstm_and_encoders_exercise_classifier_model.h5`** – tried second if `.keras` is missing or fails to load.

It also loads (required for Auto Classify):

- **`thesis_bidirectionallstm_scaler.pkl`**
- **`thesis_bidirectionallstm_label_encoder.pkl`**

**Other files in `models/` are not used by the app:**

| File                                         | Used by app?                      | What it is                                                                     |
| -------------------------------------------- | --------------------------------- | ------------------------------------------------------------------------------ |
| `final_forthesis_... .keras`                 | **Yes** (first choice)            | Full model saved by training script (Keras 3 format).                          |
| `final_forthesis_... .h5`                    | **Yes** (if .keras not used)      | Full model saved by training script (H5 format).                               |
| `thesis_bidirectionallstm_scaler.pkl`        | **Yes**                           | Feature scaler (fit on training data).                                         |
| `thesis_bidirectionallstm_label_encoder.pkl` | **Yes**                           | Class names (e.g. push-up, squat, …).                                          |
| `best_bilstm_weights.weights.h5`             | **No**                            | Training checkpoint (weights only); used only during training, not by the app. |
| `best_cnn1d_weights.weights.h5`              | **No**                            | Same for 1D CNN training.                                                      |
| `train_info.json`                            | **Yes** (sidebar + Auto Classify) | Source, architecture, test accuracy, optional class mapping.                   |
| `train_metrics.txt`                          | **Yes** (sidebar)                 | Shown as “Current model (test acc.)” in sidebar.                               |

**How to know which one is loaded:** In the app sidebar, **Model status** now shows **“Model file: …”** (e.g. `.keras` or `.h5`) so you can see which file was actually loaded.

---

## Short answer

- **The app never trains.** It only **loads** the model (and scaler, label encoder) from the **`models/`** folder.
- The model you see in the app is **whatever is currently in `models/`**. That is either:
  1. **Pre-existing** – came with the repo (if the repo included a pre-trained model), or
  2. **Trained by you** – when you ran one of the training scripts; in that case the **data** used depends on which script you ran (see below).

---

## Does training use data in the `/data` folder?

It **depends which pipeline you run**:

| What you run                             | Uses `/data`?                | What data is used                                                                                                                                                                                                                                                                                                                            |
| ---------------------------------------- | ---------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **`run_demo_training.py`**               | **No**                       | **Synthetic demo data only** (generated by `generate_demo_data.py`). Nothing from `/data` is used.                                                                                                                                                                                                                                           |
| **`run_full_pipeline.py`** (with Kaggle) | **Yes**                      | Downloads videos to `data/kaggle_exercise/`, then extracts features → sequences → trains. So training uses **data from `/data`** (Kaggle videos).                                                                                                                                                                                            |
| **`run_full_pipeline.py --no-kaggle`**   | **Only if there are videos** | If `data/real_dataset/` (or the direct-download folder) contains **.mp4/.avi/.mov** videos, those are used. If there are **no videos** (e.g. only .npy and .csv like MM-Fit), the script **does not use** those files; it falls back to **demo data** and trains on that. So in that case training **does not** use the contents of `/data`. |

So:

- **If your `/data` folder has only .npy and .csv** (e.g. from MM-Fit): the current training scripts **do not use** those for training. They need **videos** or **pre-built sequences** (from our feature pipeline or from demo).
- **If your `/data` has videos** (e.g. in `data/kaggle_exercise/`): then `run_full_pipeline.py` (Kaggle path) **does** use data from `/data` to train.

---

## How do I know what my current model was trained on?

- In the app **sidebar**, under **Model status**, you see **Source:** …
  - **`demo`** → model was trained with **synthetic demo data** (no `/data` videos used).
  - **`kaggle`** → model was trained from **Kaggle videos** (data in `data/kaggle_exercise/`).
  - **`direct`** → model was trained from **videos** you got via direct download (e.g. in `data/real_dataset/` if there were videos).
  - **`pre-trained / unknown`** → model was loaded from `models/` but no `train_info.json` was found (e.g. pre-existing or from an older run).

This **Source** is written when you run the training script (see `models/train_info.json`).

---

## Summary

- **App** = always loads from **`models/`** (no training).
- **Training** = uses **either** data from **`/data`** (only when there are **videos** in the pipeline) **or** **demo/synthetic data** (no `/data`).
- Your **sidebar “Source”** tells you what the **current** model in `models/` was trained on.
